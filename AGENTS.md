# Agent Instructions — The Nullshore Expanse Campaign

## Project Structure
- This project contains multiple campaigns. Each subfolder in the project root is its own self-contained campaign (e.g. `nullshore_expanse/`).
- All campaign-specific files (lore, draft, plot structure, etc.) live inside that campaign's subfolder.

## Pacing
- Maintain a target play time of 4 hours when editing the campaign.

## Lore Continuity
- When creating or editing the campaign story, always check the campaign's `lore.md` (located in the campaign subfolder) for continuity and logic.
- If a lore gap is found, call it out explicitly so it can be filled in before proceeding.

## D&D Beyond Compatibility
This campaign is run on D&D Beyond. Apply the following constraints at all times:

- Stat blocks must follow 5e stat block structure as used by D&D Beyond.
- Encounters must be compatible with the D&D Beyond encounter builder.
- Magic items, spells, and conditions must use official 5e names so they can be linked or referenced in D&D Beyond.
- Any NPC or monster not available natively in D&D Beyond must be flagged as requiring homebrew entry.
- Do not design mechanics, features, or systems that D&D Beyond cannot represent.
- If a design element would be difficult or impossible to implement in D&D Beyond (e.g. custom environmental rules, non-standard action economy, phased bosses with separate stat blocks), emit a warning and let the user decide how to handle it.
